1. Introduction & Overview

Linux is a robust, multi-user operating system that has become the backbone of many modern servers, cloud infrastructures, and embedded systems. Whether you’re a beginner or an experienced administrator, mastering Linux administration is essential for managing systems efficiently and securely.

In this guide, we will cover all aspects of Linux administration—from initial installation and basic command-line usage to advanced topics like storage management, network configuration, security hardening, and performance monitoring.

Note: This document is structured to help you gradually progress in your knowledge, with code examples and practical tips throughout each section.

2. Linux Installation & Setup

Linux installation can vary depending on your distribution. Whether you’re installing Ubuntu, CentOS, Debian, or any other flavor, the principles remain the same: boot the installation media, configure partitions, set up networking, and complete the basic system installation.

2.1 Booting from Installation Media

Most Linux distributions offer a live CD/USB mode that lets you test the system before installing. Insert your bootable media and reboot your system. Ensure that your BIOS/UEFI is configured to boot from USB or CD/DVD.

2.2 Partitioning and Filesystems

During installation, you will need to partition your disk. Here’s an example using fdisk to create partitions on a new disk: 

# List disks
fdisk -l

# Start partitioning /dev/sdb
fdisk /dev/sdb

# In fdisk, type:
# n (to create a new partition)
# p (for primary partition)
# Accept default values or specify sizes as needed.
# Then type:
# w (to write changes)

After partitioning, you might create a filesystem using mkfs: 

# Create an ext4 filesystem on /dev/sdb1
mkfs.ext4 /dev/sdb1

2.3 Basic System Setup

Once Linux is installed, update your system and configure basic settings. For example, on Debian/Ubuntu: 

# Update package list and upgrade packages
sudo apt update && sudo apt upgrade -y

And on CentOS/RHEL: 

# Update system packages
sudo yum update -y

3. Basic Linux Commands

Mastering basic commands is the foundation of Linux administration. Here are some essential commands:

  • ls – List directory contents.
  • cd – Change directory.
  • pwd – Print working directory.
  • cp – Copy files and directories.
  • mv – Move or rename files.
  • rm – Remove files or directories.
  • cat – Concatenate and display files.
  • grep – Search text using patterns.

Examples:

# List all files including hidden ones
ls -la

# Change to the /var/log directory
cd /var/log

# Print the current directory
pwd

# Copy a file from one location to another
cp /path/to/source.txt /path/to/destination.txt

# Recursively remove a directory
rm -r /path/to/directory

# Search for the term "error" in a log file
grep "error" /var/log/syslog

These commands form the building blocks of your daily tasks as a Linux administrator.

4. Shell Scripting Fundamentals

Shell scripting is a powerful way to automate tasks in Linux. Here we cover the basics of writing and executing shell scripts using bash.

4.1 Creating a Basic Script

Create a file called hello.sh with the following contents: 

#!/bin/bash
# A simple script to print a greeting
echo "Hello, Linux Administrator!"

Make the script executable: 

chmod +x hello.sh

Then run it: 

./hello.sh

4.2 Variables, Conditionals, and Loops

Below is an example script that demonstrates the use of variables, conditionals, and loops: 

#!/bin/bash
# Define a variable
USERNAME="admin"

# Check if the current user is the admin
if [ "$(whoami)" == "$USERNAME" ]; then
  echo "You are logged in as $USERNAME"
else
  echo "You are not $USERNAME"
fi

# Loop through a list of directories and print their contents
for DIR in /etc /var /usr; do
  echo "Contents of $DIR:"
  ls -l "$DIR" | head -n 5  # Show only first 5 entries for brevity
  echo "---------------------"
done

With these constructs, you can build more complex scripts to handle system tasks, automate backups, manage logs, and more.

5. Package Management

Linux distributions use package management systems to install, update, and remove software. The most common package managers include apt for Debian/Ubuntu and yum/dnf for CentOS/RHEL.

5.1 Using APT on Debian/Ubuntu

Basic commands using apt: 

# Update package lists
sudo apt update

# Upgrade installed packages
sudo apt upgrade -y

# Install a new package (e.g., nginx)
sudo apt install nginx -y

# Remove an installed package
sudo apt remove nginx -y

# Search for a package
apt search apache

5.2 Using YUM/DNF on CentOS/RHEL

Basic commands using yum or dnf: 

# Update the system
sudo yum update -y

# Install a package (e.g., httpd)
sudo yum install httpd -y

# Start the httpd service
sudo systemctl start httpd

# Enable the service to start at boot
sudo systemctl enable httpd

Package management is key to keeping your system secure and up to date.

6. User & Group Management

Managing users and groups is a fundamental task for Linux administrators. This includes creating and deleting users, managing groups, and setting proper file permissions.

6.1 Creating Users and Groups

To create a new user: 

# Create a user called "developer"
sudo useradd -m developer

# Set a password for the user
sudo passwd developer

To create a group and add a user to it: 

# Create a group called "devgroup"
sudo groupadd devgroup

# Add the user "developer" to "devgroup"
sudo usermod -aG devgroup developer

6.2 Managing Permissions

Linux uses read, write, and execute permissions for the user, group, and others. For example: 

# Change the permissions of a file
chmod 755 /path/to/script.sh

# Change the ownership of a file
sudo chown developer:devgroup /path/to/file.txt

Using proper permissions helps secure your system by ensuring that only authorized users can access or modify files.

7. Process & Service Management

Efficient process management ensures that your system runs smoothly. Linux provides several tools to manage processes and services.

7.1 Process Management

Some common commands for process management are: 

# List running processes
ps aux

# Monitor processes in real time
top

# Kill a process by its process ID (PID)
sudo kill -9 1234

7.2 Service Management with Systemd

Most modern Linux distributions use systemd for service management. Here are some basic systemctl commands: 

# Check the status of a service (e.g., nginx)
sudo systemctl status nginx

# Start a service
sudo systemctl start nginx

# Stop a service
sudo systemctl stop nginx

# Restart a service
sudo systemctl restart nginx

# Enable a service to start at boot
sudo systemctl enable nginx

# Disable a service from starting at boot
sudo systemctl disable nginx

By mastering process and service management, you can ensure that applications run reliably on your system.

8. Disk & Storage Management

Disk management is a critical skill for a Linux administrator. This includes partitioning disks, creating filesystems, mounting filesystems, and using logical volume management (LVM).

8.1 Partitioning and Filesystem Creation

Using fdisk or parted can help you manage partitions. For example, using fdisk: 

# List available disks
sudo fdisk -l

# Create or modify partitions on /dev/sda
sudo fdisk /dev/sda

After partitioning, create a filesystem: 

# Create an ext4 filesystem on /dev/sda1
sudo mkfs.ext4 /dev/sda1

8.2 Mounting Filesystems and /etc/fstab

To mount a filesystem: 

# Create a mount point
sudo mkdir -p /mnt/data

# Mount the partition
sudo mount /dev/sda1 /mnt/data

To ensure the partition is mounted on boot, add an entry in /etc/fstab: 

/dev/sda1   /mnt/data   ext4   defaults   0   2

8.3 Logical Volume Management (LVM)

LVM allows flexible disk management. Below is an example workflow: 

# Create a physical volume
sudo pvcreate /dev/sdb1

# Create a volume group called "vg_data"
sudo vgcreate vg_data /dev/sdb1

# Create a logical volume called "lv_files" with 10G size
sudo lvcreate -n lv_files -L 10G vg_data

# Create a filesystem on the logical volume
sudo mkfs.ext4 /dev/vg_data/lv_files

# Mount the logical volume
sudo mkdir -p /mnt/lv_files
sudo mount /dev/vg_data/lv_files /mnt/lv_files

With LVM, you can easily resize volumes or add new disks to your storage pool.

9. Network Administration

Network configuration is crucial for connecting your Linux server to other systems and the internet. This section covers basic network configuration, troubleshooting, and firewall management.

9.1 Basic Network Commands

Common commands include: 

# Display network interfaces
ip addr

# Display routing table
ip route

# Test network connectivity
ping google.com

# Display open ports and connections
netstat -tulpn

9.2 Configuring Network Interfaces

On many distributions, network interfaces are configured via files (e.g., /etc/network/interfaces on Debian/Ubuntu or /etc/sysconfig/network-scripts/ifcfg-eth0 on CentOS/RHEL). Here’s an example for Debian/Ubuntu: 

# /etc/network/interfaces
auto eth0
iface eth0 inet static
  address 192.168.1.100
  netmask 255.255.255.0
  gateway 192.168.1.1

9.3 Firewall Configuration

Linux systems often use iptables or more modern tools like ufw (Uncomplicated Firewall) to manage firewall rules. For example, using ufw: 

# Allow SSH
sudo ufw allow ssh

# Allow HTTP and HTTPS
sudo ufw allow http
sudo ufw allow https

# Enable the firewall
sudo ufw enable

# Check firewall status
sudo ufw status verbose

These commands help protect your system from unwanted network traffic.

10. Security Best Practices

Securing your Linux system is critical. This section covers a range of security topics including file permissions, SSH hardening, SELinux/AppArmor, and firewall rules.

10.1 Securing SSH

Edit the SSH configuration file (/etc/ssh/sshd_config) to improve security. For example: 

# Disable root login
PermitRootLogin no

# Change the default SSH port (optional)
Port 2222

# Limit authentication attempts
MaxAuthTries 3

After editing, restart the SSH service: 

sudo systemctl restart sshd

10.2 File and Directory Permissions

Use chmod and chown to ensure that files have the correct permissions: 

# Set file permissions to 640 and directory permissions to 750
chmod 640 /path/to/file
chmod 750 /path/to/directory

# Change the owner and group of a file
sudo chown user:group /path/to/file

10.3 SELinux and AppArmor

Many distributions use SELinux (CentOS/RHEL) or AppArmor (Ubuntu) for additional security controls.

For SELinux: 

# Check SELinux status
sestatus

# Set SELinux to permissive mode temporarily
sudo setenforce 0

For AppArmor: 

# Check AppArmor status
sudo apparmor_status

# Reload AppArmor profiles
sudo systemctl reload apparmor

Always follow best practices and keep your systems updated to protect against vulnerabilities.

11. Monitoring & Logging

Monitoring system performance and logging events are essential for maintaining the health and security of your Linux server.

11.1 System Monitoring Tools

Linux provides several tools to monitor system performance:

  • top or htop – For monitoring processes and resource usage.
  • vmstat – For memory and process statistics.
  • iostat – For input/output statistics.
  • sar – For historical system activity reports.
# Example using vmstat
vmstat 5

# Using iostat for disk I/O stats
iostat -xz 5

11.2 Logging with Syslog and Journalctl

Traditional systems use syslog to manage logs, while modern systems use systemd-journald. To view logs: 

# View system logs using journalctl
sudo journalctl -xe

# View kernel messages
sudo dmesg | tail -n 20

11.3 Log Rotation

Log files can grow quickly, so log rotation is important. logrotate is commonly used to rotate, compress, and remove old log files. Here is an example configuration snippet for /etc/logrotate.d/myapp: 

/var/log/myapp/*.log {
    daily
    missingok
    rotate 14
    compress
    delaycompress
    notifempty
    create 0640 root adm
}

Proper log management helps ensure that you can diagnose issues while keeping disk usage in check.

12. Advanced Topics

Once you’ve mastered the fundamentals of Linux administration, you may wish to explore advanced topics such as automation, containerization, and kernel tuning.

12.1 Automation with Ansible

Ansible is an automation tool that allows you to manage configurations across multiple servers. Below is an example playbook that installs and starts Nginx: 

---
- name: Install and Start Nginx
  hosts: webservers
  become: yes
  tasks:
    - name: Install Nginx
      apt:
        name: nginx
        state: present
      when: ansible_os_family == "Debian"

    - name: Start Nginx service
      service:
        name: nginx
        state: started
        enabled: yes

12.2 Containerization with Docker

Containers are a modern way to deploy applications. Here’s how to install Docker on Ubuntu: 

# Update existing package index and install prerequisites
sudo apt update
sudo apt install apt-transport-https ca-certificates curl gnupg-agent software-properties-common -y

# Add Docker’s official GPG key
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

# Set up the stable repository
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"

# Install Docker Engine
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io -y

# Verify Docker installation
sudo docker run hello-world

12.3 Kernel Tuning

Optimizing kernel parameters can improve system performance. Parameters are usually set in /etc/sysctl.conf or files under /etc/sysctl.d/. For example, to increase the maximum number of open files: 

# Edit /etc/sysctl.conf and add:
fs.file-max = 100000

# Apply the changes immediately
sudo sysctl -p

Kernel tuning is a powerful tool when you need to optimize performance for high-demand applications.
